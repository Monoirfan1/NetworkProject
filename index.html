VLAN 10 Gateway Connectivity Test

Verify Guest WiFi Network Access

What is VLAN 10?

VLAN stands for "Virtual Local Area Network." Think of it like creating separate, invisible networks within your physical network. VLAN 10 is specifically designated for the Guest WiFi network at Hansa Villa hotel. When guests connect their laptops, tablets, or phones to the hotel WiFi, they're automatically placed into VLAN 10. This network is completely separate from the hotel's operations network, which means guests cannot accidentally (or intentionally) access sensitive hotel systems like the front desk computer, reservation system, or security cameras. VLAN 10 gives guests internet access while keeping the hotel's internal systems secure. The gateway at 10.10.10.1 is the "doorway" that all guest traffic must pass through to reach the internet.

Why This Test Matters

This is the most important test in your entire verification process. Here's why: if the VLAN 10 gateway doesn't respond, guests cannot use WiFi at all. Imagine checking into a hotel and being told "the WiFi is down" - that's a terrible experience. This test confirms that the gateway is online, responding instantly (under 1 millisecond), and ready to handle guest traffic. When you run this ping test and see 100% success, you're proving that Router0's sub-interface for VLAN 10 (called Gig0/0.10) is configured correctly, is powered on, and is actively routing traffic. If this test fails, every guest device will be unable to access the internet. Success here means the foundation of your guest network is solid.

What This Command Does (Detailed Explanation)

The command "ping 10.10.10.1" is a network diagnostic tool that tests whether a device is reachable and responding. Here's what happens when you run this command:

Step 1: Your command tells Router0 to send a small test message (called an "ICMP echo request") to the IP address 10.10.10.1, which is the VLAN 10 gateway.

Step 2: The gateway receives this message and immediately sends back a reply (called an "ICMP echo reply") saying "I'm here and working!"

Step 3: This process repeats 5 times automatically (you'll see 5 replies).

Step 4: Router0 calculates how many replies it received versus how many it sent, and shows you a success percentage.

Think of it like sending someone 5 text messages asking "Are you there?" and counting how many "Yes!" replies you get back. If you get 5 out of 5 replies, that's 100% success. If you get 0 replies, that means the gateway is offline, unreachable, or not configured. The response time (shown as "time<1ms") tells you how fast the reply came back - under 1 millisecond is instant and excellent. This speed confirms there are no network delays or congestion issues.

Why IP address 10.10.10.1? This is the default gateway for VLAN 10. Every device on the guest network (like a guest's laptop with IP 10.10.10.50) sends all its internet traffic to this gateway first. The gateway then routes that traffic to the internet. If this gateway doesn't work, no guest device can access anything beyond the local network.

Problem 1: "Request timed out" or "Success rate is 0 percent (0/5)" or "100% packet loss"

What this error means: You sent 5 ping requests but got ZERO responses back. The gateway at 10.10.10.1 is not responding at all. It's like calling someone's phone and it just rings forever - nobody picks up.

Why this happens:

  • The VLAN 10 interface (Gig0/0.10) might not exist on Router0
  • The interface might be turned off (administratively shutdown)
  • The physical cable connecting Router0 to Switch0 might be unplugged
  • Router0 might be powered off (unlikely in Packet Tracer)
  • The interface might have the wrong IP address configured

How to diagnose step-by-step:

  1. First, check your prompt. Do you see "Hansa_Router#" with the # symbol? If you see "Hansa_Router>" you need to go back to the password section above and type "enable" then password "Class123".
  2. Once you have the Hansa_Router# prompt, type this command: show ip interface brief and press ENTER.
  3. Look at the output carefully. Find any line that contains "10.10.10.1" in the IP-Address column.
  4. If you see a line with 10.10.10.1 and Status shows "up" and Protocol shows "up" - this means "up/up" and the interface is working. The problem is elsewhere (maybe a routing issue).
  5. If you see a line with 10.10.10.1 but Status shows "administratively down" - this means the interface exists but is turned off. You need to enable it.
  6. If you DON'T see 10.10.10.1 anywhere in the output - this means the VLAN 10 interface was never created. It doesn't exist.

How to fix:

  • If interface is administratively down: Type configure terminal, then interface GigabitEthernet0/0.10, then no shutdown, then end. Try the ping again.
  • If interface doesn't exist: Contact your instructor - the VLAN 10 sub-interface needs to be created and configured.
  • If you're still stuck: Verify the parent interface (Gig0/0) is up by typing show ip interface brief and looking for Gig0/0 with status "up/up".

Problem 4: "Destination Host Unreachable"

What this error means: The router received your ping command successfully, but it can't figure out how to route the traffic to reach 10.10.10.1. The destination exists in the configuration, but there's no path to get there.

üîß Comprehensive Troubleshooting Guide

Common Issues & Solutions

All pings fail immediately after opening file

Cause: Switch ports are still in orange (STP convergence not complete)

Solution: Wait 2-5 minutes for all ports to turn green. Do not proceed until GREEN.

"Invalid input detected" error on every command

Cause: Not in privileged mode OR typo in command

Solution: Type enable and password Class123. Use Copy buttons to avoid typos.

show vlan brief returns "Invalid input"

Cause: Command run on Router0 instead of Switch0

Solution: Close Router0 CLI, open Switch0 CLI, enable, then retry command.

ACLs show empty or missing

Cause: ACL configuration was not saved or not applied to interfaces

Solution: Verify ACL configuration in running-config. Check interface ACL assignments.

Some interfaces show "administratively down"

Cause: Interface is shutdown (disabled)

Solution: Enter interface config mode and type no shutdown to enable.

Paste doesn't work in CLI

Cause: Packet Tracer paste requires specific method

Solution: Copy text, then RIGHT-CLICK in CLI window and select Paste (not Ctrl+V).
